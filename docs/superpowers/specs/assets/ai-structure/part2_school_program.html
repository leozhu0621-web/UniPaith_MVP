TWO WAYS IN & QUANTIFYAPI

Way ① · We build it — the crawler (unclaimed programs)

  • learns from all public materials — governed crawl + verified feeds · grounded · never invents · cites source
  • we determine both: attributes (outcomes · selectivity · cost · funding · accreditation · format · requirements · photos) and preferences (the target applicant)
  • source changes → re-enter the loop (consent + cap) · stops the moment the school claims

Way ② · Claimed school users (first-party, authoritative)

  • claim = verify ownership → unlocks editing, crawler steps away
  • from then on they fully own:

Attributes

cost · outcomes · deadlines · capacity · requirements (direct entry · CSV · docs)

Preferences

target applicant profile + the importance weights that drive recruiting

QUANTIFY — raw (crawled HTML · form · CSV cell · doc) → typed value, same Prompt-Library schema:

Categorical (degree · field · country)→ enum (taxonomy)
Numeric (cost · admit rate · outcomes)→ number
Range (cost band · GPA / test band)→ band
Boolean (funding · part-time · online)→ flag
Date (deadlines · terms)→ ISO date
