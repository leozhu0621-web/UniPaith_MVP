# 17 · Inbox

> Application-threaded conversations + platform notifications. Each thread links to the relevant application and to related calendar items. Separates human messages from system updates. Lives at `/s/manage?tab=messages`.
>
> Status: **draft v1.0** · 2026-05-29 · Route: `/s/manage?tab=messages` (+`&thread=:id`).

---

## 1. Purpose

Centralize admissions-related conversations. Convert message requests into actionable items linked to the right checklist item or calendar event so the student never loses context.

## 3. Filters

- **By application** — only this app's threads.
- **By message type** — human vs system.
- **By action state** — needs reply / requested / completed / status update only.

Sort by most urgent (default), most recent, most action-required.

---

## 4. Thread types

### 4.1 Human threads
From admissions officers / interview coordinators / recommendation processors. Threaded by application.

### 4.2 System messages
- Missing-item alerts.
- Requirement reminders.
- Submission confirmations.
- Status updates.
- AI run notifications ("Your match scores updated after profile edit").

System messages visually separated; collapsible.

---

## 5. Action labels

Each thread carries one of:
- **Needs reply** — student should respond.
- **Document requested** — student should attach.
- **Clarification required** — student should answer.
- **Interview invite** — student should accept/decline/reschedule.
- **Status update only** — informational.

Each thread shows: who is waiting on whom (student vs school) + due date if any.

---

## 6. Attachments and request handling

Messages that request materials provide direct actions:
- **Attach a document** (from existing student materials or upload).
- **Submit a link or portfolio item**.
- **Confirm completion**.

Each request links to the relevant checklist item in `15-applications.md` so completion stays consistent.

---

## 7. AI-suggested replies (NEW — per `45` §13)

When a thread is `needs_reply` or `clarification_required`, an "AI assist" suggested-reply card appears below the thread.
- Student edits before sending. Never sends automatically.
- Up to 2 alternate drafts (tones).
- Generated by `InboxReplyDrafter` agent.

If `consent.matching=false` OR agent fails: no card; student types from scratch.

## 8. Data shape

```ts
type Thread = {
  id: string;
  application_id: string;
  application: { program_name: string; institution_name: string };
  type: 'human' | 'system';
  subject: string;
  participants: Array<{ id: string; role: 'student' | 'admissions_officer' | 'system'; name: string }>;
  action_label: 'needs_reply' | 'document_requested' | 'clarification_required' |
                'interview_invite' | 'status_update_only' | 'completed';
  due_date: ISO8601 | null;
  waiting_on: 'student' | 'school' | 'none';
  unread: boolean;
  last_message_at: ISO8601;
  messages: Message[];
  linked_checklist_item_id: string | null;
  linked_calendar_item_id: string | null;
};

type Message = {
  id: string;
  thread_id: string;
  sender: 'student' | 'admissions_officer' | 'system';
  body: string;
  attachments: Attachment[];
  sent_at: ISO8601;
  read_at: ISO8601 | null;
  status: 'sent' | 'delivered' | 'read';
};
```

Endpoints:
- `GET /me/inbox/threads?filter=...`.
- `GET /me/inbox/threads/:id`.
- `POST /me/inbox/threads/:id/messages` — body: `{body, attachments}`.
- `POST /me/inbox/threads/:id/mark-complete` — sets action_label to `completed`.
- `POST /me/inbox/threads/:id/suggested-reply` — calls `InboxReplyDrafter`.

## 9. States

- **Empty:** "No conversations yet. Schools will reach out as you start applications."
- **Loading:** skeleton list + thread skeleton.
- **No selected thread:** "Pick a conversation to see it here."
- **AI suggestion failure:** card hidden silently; student types.
