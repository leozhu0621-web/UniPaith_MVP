## 1. Purpose

The Master Paper splits Stage 2 into two parts:
1. **Broad strategy** — career → degree → academic / financial / geographic narrative. The "what should I aim for?" answer.
2. **Specific shortlist** — ranked programs with fitness + confidence scores and plain-language rationale.

Both surface at `/s/explore`. The Strategy view sits at the top; the search/browse engine fills the rest of the page (`10-discovery.md`).

Each program card uses the canonical display card schema (`02-design-system.md` §5). Match-context fields populated:

## 4A. Probability bands (admit / scholarship / waitlist)

**Distinct from fitness + confidence.** Fitness = how well the program fits the student; confidence = how sure the model is. **Probability bands** answer a different question: *"what's my realistic shot?"* — conservative ranges for the three outcomes. From `Feature List V1` (a `49` §2 MVP-extend, sourced from the OUTPUT schema `42` §4.12 `aid_scholarship_likelihood_band` + admit-likelihood + history data).

### Display
On the card (expandable) and prominently on the program detail page (`11`):
```
Your realistic shot
  Admit        ▓▓▓▓▓░░░░░  35–50%   (target)
  Scholarship  ▓▓▓░░░░░░░  15–25%
  Waitlist     ▓▓░░░░░░░░  ~10%
```
- Always a **range**, never false-precision; width encodes uncertainty.
- Band label: likely / target / reach / unlikely.
- Tap → AI Rationale Popover (`02` §6): top drivers (test fit, GPA vs class profile, capacity, historical admit rate from `42` §3 history data).

### Rules
- Computed only when the program has enough historical signal AND the student meets match-ready minimum (`42` §6.1); else show "Not enough data yet" rather than a misleading band.
- **Honesty guardrail:** never present a probability as a guarantee; the band is decision-support, not a promise — institutions decide, not the model (`46` §6). Subject to fairness review.
- Recompute on profile-version / program-version change (same cache-key family as matches).

## 5. Iterative refinement

Two refinement vectors:

### 5.1 Constraint chips
Edit / remove chips → search re-runs → results update live. Detailed in `10-discovery.md` §4.

### 5.2 Priority sliders
A "Refine priorities" link below the chips opens a sheet with:
- Cost importance · Outcomes importance · Selectivity importance · Location importance · Modality importance · Time-to-degree importance
- 6 sliders 0–10.

Save → updates `student_profiles` priority weights → re-runs match.

---

## 6. Grouping / banding

Default grouping: three bands (reach / target / safer). Toggle to flat ranked list.

Band thresholds (per profile + strategy + program selectivity):
- **Reach:** fitness ≥ 50, selectivity > student's stated tolerance.
- **Target:** fitness ≥ 65, selectivity ≈ tolerance.
- **Safer:** fitness ≥ 75, selectivity < tolerance.

## 7. Data shape

```ts
type MatchResult = {
  program_id: string;
  fitness_score: number;       // 0-100
  confidence_score: number;    // 0-100
  band_label: 'reach' | 'target' | 'safer';
  matching_signals: Array<{ category: string; signal: string; weight: number }>;
  probability_bands: {                      // §4A — null when insufficient data
    admit: { low: number; high: number; label: 'likely'|'target'|'reach'|'unlikely' };
    scholarship: { low: number; high: number } | null;
    waitlist: { approx: number } | null;
    drivers: Array<{ signal: string; direction: 'up'|'down' }>;
  } | null;
  generated_at: ISO8601;
  rationale_cached_at: ISO8601 | null;
};

type ExploreState = {
  active_strategy: Strategy | null;
  match_results: MatchResult[];
  search_query: string;
  chips: ConstraintChip[];
  filters: FilterState;
  band_grouping: boolean;
};
```

Backend endpoints:
- `GET /me/matches?refresh=...` — returns ranked matches.
- `POST /me/matches/:program_id/explain` — rationale + cached.
- `POST /me/strategy/generate` — runs StrategyAgent.
- `GET /me/strategy/active` — returns active strategy.

## 14. Open questions

- **Defaulting reach/target/safer mix.** Spec defaults? Per Master Paper, students set their own stretch/target/safety mix preference; absent that, default to 2/4/2 distribution.
- **Sorting within band.** Current code sorts by fitness desc; consider a "secondary sort by confidence" toggle.
- **Strategy → match link.** When strategy regenerates, should matches re-rank immediately? Yes, but cap the cost (re-rank only top 50 candidates by similarity to the new strategy embedding).
