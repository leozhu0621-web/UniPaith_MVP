## 1. Purpose

Most platforms end at submission. UniPaith doesn't ŌĆö students need help **deciding** between offers, comparing financial packages, and meeting the offer-acceptance deadlines.

---

## 2. Decision states

Per application:
- `pending` (default) ŌĆö submitted, awaiting decision.
- `accepted` ŌĆö offer received.
- `rejected` ŌĆö denied.
- `waitlisted` ŌĆö on the waitlist.
- `deferred` ŌĆö pushed to later round.
- `accepted_by_student` ŌĆö student accepted the offer.
- `declined_by_student` ŌĆö student declined the offer.
- `withdrawn` ŌĆö student withdrew the application before decision.

---

## 3. Offer capture

When an institution issues an offer (internal submission flow) OR student records receiving one (external flow), an `Offer` is created:

```ts
type Offer = {
  id: string;
  application_id: string;
  decision_date: ISO8601;
  offer_type: 'full_admission' | 'conditional' | 'waitlist_to_admit' | 'partial' | 'transfer_credit_offer';
  response_deadline: ISO8601;
  conditions: string | null;                  // e.g., "complete prerequisites by Aug 2027"
  scholarship_amount: number | null;
  scholarship_currency: string | null;
  tuition_estimate: number | null;
  total_cost_estimate: number | null;
  start_term: { season: string; year: number };
  next_step_actions: Array<{ action: string; by_date: ISO8601 }>;
  attached_letter: file_ref | null;
  plain_language_brief: PlainLanguageBrief | null;
};
```

`PlainLanguageBrief` generated by `OutcomeBriefForOfferLetter` (`45-ai-agents-claude.md` ┬¦15).

## 5. Decision comparison

Modal/page that compares all current offers side-by-side. Dimensions:

- Cost: tuition, scholarship, net cost.
- Fit: original fitness + confidence + post-decision sentiment if captured.
- Outcomes: program's salary bands + placement.
- Location.
- Constraints: anything the student flagged as a must-have.
- Decision timeline: response deadlines.

Highlights "best value" / "best fit" / "most affordable" indicators.

---

## 6. Post-submission workflow continuation

After the student accepts an offer:
- Status flips to `accepted_by_student`.
- Other pending applications can be set to `withdrawn` in bulk (with a "Are you sure?" confirmation).
- A milestone reminder is created: "Submit deposit" / "Schedule orientation" / "Apply for housing".
- The Apply stage transitions visually into a "You're in" celebration card on the dashboard.

---

## 7. Data shape

Endpoints:
- `POST /me/applications/:id/offers` ŌĆö record receipt of an offer (external mode) or auto-created on internal decision.
- `PATCH /me/applications/:id/offers/:offer_id` ŌĆö student response (accept/decline).
- `GET /me/offers/comparison` ŌĆö current offers comparison data.
- `POST /me/applications/:id/withdraw`.
