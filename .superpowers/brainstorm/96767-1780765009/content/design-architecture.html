Design · Section 3 — how it maps to the codebase

The data model, extractor, matching, and strategy flow all stay. We replace the Discover UI and retrain the conversation agent into Uni. One real backend change: extraction stops being gated by a fixed "track."

Frontend

KEEP
• Chat plumbing (TanStack Query, session/message API)
• Message send + streaming-dots states
• The Explore/match surface ("See programs")
• Profile page as the durable record
REPLACE
• DiscoverHomePage → single-column, Uni header, no rails
• ChatPanel → Uni conversation (warm bubbles, one-at-a-time, counselor quick-replies)
• EmptyState → Uni's warm opening (one question, not a prompt wall)
REMOVE (from Discover)
• TrackSelector (Profile/Goals/Needs tabs)
• LayerSwitcher (basic/personality/identity)
• ReadinessRail + ArtifactRail (main view)
• StrategyHandoffCTA (50%-gate bar)
ADD
• Inline "✓ Noticed…" cards (editable)
• Living-profile drawer/view ("✦ Your profile ▸")
• In-thread match-handoff card
• Uni persona (avatar "U", name, voice)

Backend

KEEP (data model unchanged)
• discovery_sessions / discovery_messages
• student_goals / student_needs / student_identity
• The extractor, validator, judge, matching, strategy
• Feature flags + rule-based fallback
CHANGE
• orchestrator.py → Uni counselor playbook system prompt (always/never)
• Extraction routing: by content, not session.track — one convo feeds goals+needs+identity
• Readiness/handoff surfaced as Uni's offer, not a % gate
• One unified session (track-less / "discovery")
Key decision · data flow
One Uni conversation  →  extractor (routes each turn by content)  →  goals · needs · identity
    ↓ read                                 ↓ read
"✦ Your profile" (warm, editable)        matching engine → "See programs that fit me" → Explore
Agent training (the part you emphasized)

Uni's "always/never" counselor playbook becomes her system prompt and a constitution + eval set — reusing the existing chatbot-eval / constitution infrastructure (§61/§62) so we can measure that Uni behaves like a real counselor (asks open questions, reflects, one-at-a-time, no slang/interrogation) before shipping. Rule-based fallback stays so a flag-off or LLM error never breaks Discovery.
